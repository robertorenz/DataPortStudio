DataPortStudio — Guía del usuario

Un administrador de bases de datos estilo Navicat para SQL Server, SQLite, MySQL/MariaDB, Firebird, MongoDB y archivos Clarion TPS/DAT.

Descripción general

DataPortStudio le permite conectarse a varios tipos de bases de datos, explorar su estructura en un árbol, abrir tablas y ver & editar registros directamente. Puede ejecutar SQL, construir consultas de forma visual, diseñar tablas, copiar tablas entre bases de datos (incluso entre motores distintos), importar/exportar datos y administrar usuarios y roles.

Cada conexión muestra el icono de su motor para distinguir sus bases de datos de un vistazo.

Primeros pasos

  1. DataPortStudio es un único DataPortStudio.exe portátil — no requiere instalación.
  2. En el primer inicio el paquete comprimido se autoextrae (un par de segundos); los siguientes inicios son rápidos.
  3. Agregue su primera conexión desde Conexión ▸ Agregar conexión (o la barra de herramientas).
  4. Expándala en el árbol de la izquierda para explorar bases de datos y tablas; haga doble clic en una tabla para abrirla.
Idioma: DataPortStudio está disponible en inglés y español. Cambie el idioma en Archivo ▸ Configuración. Esta guía se abre en el idioma que tenga seleccionado.

Conexiones

Use Conexión ▸ Agregar conexión para abrir el editor de conexiones. Asigne un nombre, elija el tipo de base de datos, complete los campos de ese motor y haga clic en Probar para verificar antes de guardar. Para cambiar o eliminar una conexión más adelante, selecciónela en el árbol y use Conexión ▸ Editar / Eliminar (o el clic derecho).

  • SQL Server — Autenticación de Windows o inicio de sesión SQL; constructor por campos o cadena de conexión sin procesar.
  • Probar conexión confirma que la configuración funciona antes de guardar.
  • Las conexiones se guardan entre sesiones; las contraseñas se protegen con la protección de datos de Windows.

Árbol y pestaña Objetos

La barra lateral izquierda muestra sus conexiones como un árbol. Expanda una conexión para llegar a sus bases de datos, esquemas y carpetas de objetos (Tablas, Vistas, Funciones, Procedimientos). Use el cuadro de filtro en la parte superior para localizar tablas por nombre.

Al seleccionar una carpeta Tablas, un esquema o una base de datos se abre la pestaña Objetos: una lista estilo Navicat con el conteo de filas cuando está disponible. En las conexiones Clarion (TPS/DAT), al seleccionar la conexión se listan sus archivos (con tamaño y fecha) en la pestaña Objetos. Haga doble clic en cualquier elemento para abrirlo.

Ver y editar registros

Haga doble clic en una tabla (o use Abrir en el menú contextual) para cargar sus filas en una cuadrícula.

  • Edite una celda escribiendo en ella; los cambios se resaltan hasta que guarde.
  • Agregue una fila en el marcador inferior, o elimine las filas seleccionadas.
  • Guardar escribe los cambios pendientes en la base de datos; Recargar los descarta.
  • Filtre y ordene columnas desde la barra de herramientas; el SQL en vivo se muestra en el panel SQL.
  • El panel Detalle de celda muestra el contenido completo de celdas grandes; el panel Estructura muestra las columnas, el DDL y los índices de la tabla.
Tablas sin clave primaria: DataPortStudio pregunta qué columna(s) identifican de forma única una fila (se usan en la cláusula WHERE al editar/eliminar). Elija con cuidado, o mantenga la opción predeterminada "todas las columnas".

Un límite de filas (configurable en Configuración) limita cuántas filas se cargan a la vez, para abrir tablas grandes con rapidez.

Fechas y horas Clarion

Las aplicaciones Clarion guardan fechas y horas como números enteros (una Fecha estándar son los días desde el 28 dic 1800; una Hora estándar son centésimas de segundo desde la medianoche). DataPortStudio detecta automáticamente esas columnas y puede mostrarlas como fechas/horas legibles en lugar de números — active/desactive la presentación desde la barra de la tabla cuando se detecten columnas Clarion.

Archivos Clarion TPS y DAT

DataPortStudio lee los archivos de datos Clarion directamente — sin controlador ODBC ni runtime.

Configurar una conexión

  1. Agregue una conexión y elija TPS (TopSpeed .tps) o Clarion DAT (clásico .dat).
  2. Elija la carpeta que contiene los archivos. Cada archivo de la carpeta aparece como una tabla.
  3. Haga clic en Probar para confirmar que la carpeta tiene archivos legibles y guarde.

Explorar

Seleccione la conexión para listar sus archivos en la pestaña Objetos, o expándala en el árbol. Abra un archivo para explorar registros con filtro, orden y exportación. Las columnas de fecha/hora Clarion (LONG) se detectan y se muestran igual que en las tablas SQL.

Edición TPS: los registros TPS se abren en una cuadrícula editable — haga clic en una celda para modificarla y pulse Guardar. Los cambios se escriben de vuelta al archivo .tps binario. No se puede agregar ni eliminar filas (requeriría mantener los archivos de índice). Los archivos DAT siguen siendo de solo lectura.

TPS es el formato TopSpeed; DAT es el formato ISAM clásico de Clarion (decodificado a partir del Boletín Técnico 117 de Clarion). Los archivos de clave/índice (.K??/.I??) y los memos .MEM no se leen.

Ventana y diseñador de consultas

Archivo ▸ Nueva consulta abre un editor SQL para la conexión seleccionada. Escriba SQL, ejecútelo y explore la cuadrícula de resultados; los resultados se pueden exportar. (La ventana de consultas no aplica a MongoDB ni a los archivos Clarion.)

Archivo ▸ Diseñador de consultas abre un diseñador visual: elija tablas, columnas, agregue uniones y condiciones, y DataPortStudio genera el SQL por usted. Funciona con SQL Server, SQLite, MySQL/MariaDB y Firebird.

Diseñador de tablas

Para SQL Server y SQLite, haga clic derecho en una carpeta Tablas ▸ Nueva tabla, o en una tabla ▸ Diseñar, para definir columnas, tipos, nulabilidad y la clave primaria. Editar una tabla SQLite existente usa una reconstrucción segura de la tabla de forma interna.

Copiar y pegar tablas

Haga clic derecho en una tabla ▸ Copiar (o Ctrl+C en el árbol), luego Pegar (Ctrl+V) sobre una carpeta Tablas o una base de datos. Se le preguntará si copiar solo la estructura o estructura + datos, y la copia se nombra automáticamente si el nombre ya existe.

Puede pegar en la misma conexión, en otra conexión del mismo motor, o incluso en un motor relacional distinto (p. ej. SQLite → SQL Server). DataPortStudio asigna a cada columna un tipo compatible, crea la tabla y copia las filas. Las copias grandes a SQL Server usan transmisión masiva de alta velocidad.

Clarion → SQL: revisar y ajustar los tipos de columna

Al copiar un archivo Clarion TPS o DAT a una base de datos SQL, DataPortStudio muestra un diálogo de asignación de columnas antes de crear la tabla:

  • Cada columna de origen aparece con su tipo Clarion y un tipo SQL propuesto.
  • Elija el tipo SQL en una lista desplegable (específica del motor de destino) y defina un tamaño / precisión cuando corresponda (longitud para texto, precisión,escala para decimales).
  • Las columnas que parecen fechas/horas Clarion se preasignan a date / time de SQL, y sus valores se convierten al copiar (no se vuelcan como números crudos).
  • Use Restablecer a sugerido para volver a la asignación automática, y luego Aplicar y copiar.
¿No quiere convertir una fecha/hora? Solo vuelva a poner esa columna como int en el diálogo y se copiará el número Clarion sin procesar.

Generar INSERT, Importar y Exportar

  • Generar INSERT (clic derecho en una tabla de SQL Server) produce un script INSERT con sus filas, que puede ver, copiar o guardar como .sql.
  • Importar datos carga filas a una tabla desde un archivo.
  • Exportar la tabla actual o los resultados de una consulta a CSV, TXT, JSON, XML, HTML, Excel (XLSX / XLS heredado), DBF o un script SQL. Elija el alcance del filtro y el formato en el diálogo de exportación, y abra el archivo o su carpeta al terminar.

Usuarios y roles

El botón Usuarios de la barra de comandos abre el Administrador de usuarios para la conexión seleccionada (disponible en SQL Server y MySQL/MariaDB). Puede listar usuarios y roles, crearlos/eliminarlos, establecer o cambiar contraseñas y administrar privilegios — inicios de sesión/roles y permisos de servidor en SQL Server; cuentas user@host, privilegios globales y por base de datos, y roles en MySQL 8 / MariaDB.

Configuración

Archivo ▸ Configuración le permite elegir el idioma (inglés / español), el límite de filas predeterminado al abrir tablas y opciones de apariencia. Los cambios se aplican de inmediato.

Preguntas frecuentes

1. ¿Necesito instalar DataPortStudio o .NET?

No. DataPortStudio es un único DataPortStudio.exe portátil — solo descárguelo y ejecútelo. Es autónomo, por lo que no requiere instalar .NET. El primer inicio tarda un par de segundos mientras se autoextrae.

2. ¿Dónde se guardan mis conexiones y contraseñas?

Las conexiones se guardan en su PC y se restauran cada vez que inicia DataPortStudio. Las contraseñas se cifran con la protección de datos de Windows (vinculada a su cuenta de usuario de Windows), no se guardan en texto plano.

3. ¿Por qué no puedo agregar o eliminar filas en una tabla TPS o DAT?

Clarion TPS admite edición de celdas (UPDATE) — haga clic en una celda, cambie el valor y guarde. Agregar y eliminar filas no está soportado porque esas operaciones requieren reconstruir los archivos de índice TPS (.K??/.I??), que DataPortStudio no gestiona. Clarion DAT y MongoDB siguen siendo de solo lectura.

4. ¿Cómo cambio entre inglés y español?

Abra Archivo ▸ Configuración y elija el idioma. La interfaz se actualiza en vivo (sin reiniciar), y esta Guía del usuario se abre en el idioma que tenga seleccionado.

5. Mi tabla solo muestra parte de las filas — ¿por qué?

Las tablas se abren con un límite de filas para que las tablas grandes carguen rápido. Aumente el límite predeterminado en Archivo ▸ Configuración, o aplique un filtro para reducir a las filas que necesita.

6. ¿Cómo paso un archivo Clarion TPS/DAT a SQL Server?

Cree una conexión Clarion apuntando a la carpeta, haga clic derecho en el archivo ▸ Copiar, y luego Pegar sobre la carpeta Tablas de la base de destino. Revise los tipos SQL propuestos en el diálogo de asignación, ajústelos si lo desea y haga clic en Aplicar y copiar.

7. Mis fechas/horas Clarion aparecen como números — ¿cómo lo corrijo?

Clarion guarda fechas y horas como enteros. DataPortStudio detecta esas columnas y puede mostrarlas como fechas/horas legibles — use el interruptor en la barra de la tabla. Al copiar a SQL, esas columnas se preasignan a date/time y los valores se convierten automáticamente.

8. ¿Puedo copiar una tabla entre dos tipos de base de datos distintos?

Sí. Puede copiar entre motores relacionales (por ejemplo SQLite → SQL Server, o Firebird → MySQL) y desde archivos Clarion a cualquier base de datos SQL. DataPortStudio asigna a cada columna un tipo compatible y copia las filas.

9. Al editar una tabla advierte que "no hay clave primaria" — ¿qué elijo?

Sin clave primaria, DataPortStudio necesita saber qué columna(s) identifican de forma única una fila para que las ediciones y eliminaciones afecten la correcta. Elija una columna (o conjunto) que sea única, o mantenga la opción predeterminada "todas las columnas", que es la más segura.

10. ¿Cómo actualizo DataPortStudio?

Descargue el DataPortStudio.exe más reciente desde la página Ayuda ▸ Última versión y reemplace su copia. Sus conexiones y configuración guardadas se conservan. Puede ver su versión actual en Ayuda ▸ Acerca de.
