DataPortStudio — User Guide

A Navicat-style database manager for SQL Server, SQLite, MySQL/MariaDB, Firebird, PostgreSQL, Oracle, MongoDB and Clarion TPS/DAT files.

Overview

DataPortStudio lets you connect to several kinds of databases, browse their structure in a tree, open tables, and view & edit records in place. It can run SQL, build queries visually, design tables, copy tables between databases (even across different engines), import/export data, and manage users and roles.

Each connection is tagged with an engine icon so you can tell your databases apart at a glance.

Getting started

  1. DataPortStudio is a single portable DataPortStudio.exe — no installation is required.
  2. On first launch the compressed bundle self-extracts (a couple of seconds); later launches are fast.
  3. Add your first connection from Connection ▸ Add Connection (or the toolbar).
  4. Expand it in the left tree to browse databases and tables; double-click a table to open it.
Language: DataPortStudio is available in English and Spanish. Switch the language in File ▸ Settings. This guide opens in whichever language you have selected.

Connections

Use Connection ▸ Add Connection to open the connection editor. Give the connection a name, pick the Database type, fill in the fields for that engine, and click Test to verify before saving. To change or delete a connection later, select it in the tree and use Connection ▸ Edit / Remove (or right-click).

  • SQL Server — Windows Authentication or SQL login; a field builder or a raw connection string.
  • Test Connection confirms the settings work before the connection is saved.
  • Connections are saved between sessions; passwords are protected with Windows data protection.

Tree & the Objects tab

The left sidebar shows your connections as a tree. Expand a connection to reach its databases, schemas, and object folders (Tables, Views, Functions, Procedures). Use the filter box at the top of the sidebar to locate tables by name.

Selecting a Tables folder, a schema, or a database opens the Objects tab — a Navicat-style list of the tables with row counts where available. For Clarion (TPS/DAT) connections, selecting the connection itself lists its files (with size and date) in the Objects tab. Double-click any item to open it.

Viewing & editing records

Double-click a table (or use Open from the right-click menu) to load its rows into a grid.

  • Edit a cell by typing into it; changes are highlighted until you save.
  • Add a row at the bottom placeholder, or delete the selected row(s).
  • Save writes pending changes back to the database; Reload discards them.
  • Filter and sort columns from the toolbar; the live SQL is shown in the SQL panel.
  • The Cell detail panel shows the full content of large cells; the Structure panel shows the table's columns, DDL and indexes.
Tables without a primary key: DataPortStudio asks which column(s) uniquely identify a row (used in the WHERE clause for edits/deletes). Choose carefully, or keep the default "all columns".

A row limit (configurable in Settings) caps how many rows load at once, so large tables open quickly.

Clarion dates & times

Clarion applications store dates and times as plain integers (a Standard Date is days since 28 Dec 1800; a Standard Time is hundredths of a second since midnight). DataPortStudio automatically detects those columns and can display them as readable dates/times instead of raw numbers — toggle the display from the table toolbar when Clarion columns are detected.

Clarion TPS & DAT files

DataPortStudio reads Clarion data files directly — no ODBC driver or runtime needed.

Setting up a connection

  1. Add a connection and pick TPS (TopSpeed .tps) or Clarion DAT (classic .dat).
  2. Choose the folder that contains the files. Each file in the folder appears as a table.
  3. Click Test to confirm the folder has readable files, then save.

Browsing

Select the connection to list its files in the Objects tab, or expand it in the tree. Open a file to browse records with filter, sort and export. Clarion LONG date/time columns are detected and rendered just like in SQL tables.

TPS editing: TPS records open in an editable grid — click a cell to change it and press Save. Changes are written back to the binary .tps file. Adding and deleting rows is not supported (index-file maintenance would be required). DAT files remain read-only.

TPS is the TopSpeed format; DAT is the older classic Clarion ISAM format (decoded from Clarion Technical Bulletin 117). Key/index files (.K??/.I??) and .MEM memos are not read.

Query window & visual builder

File ▸ New Query opens a SQL editor for the selected connection. Type SQL, run it, and browse the results grid; results can be exported. (The query window doesn't apply to MongoDB or Clarion files.)

File ▸ Query Builder opens a visual designer: pick tables, choose columns, add joins and conditions, and DataPortStudio generates the SQL for you. It works with SQL Server, SQLite, MySQL/MariaDB and Firebird.

Table designer

For SQL Server and SQLite, right-click a Tables folder ▸ New Table, or a table ▸ Design, to define columns, types, nullability and the primary key. Editing an existing SQLite table uses a safe table-rebuild behind the scenes.

Copy & paste tables

Right-click a table ▸ Copy (or Ctrl+C in the tree), then Paste (Ctrl+V) onto a Tables folder or database. You'll be asked whether to copy structure only or structure + data, and the copy is auto-named if the name is taken.

You can paste into the same connection, a different connection of the same engine, or even a different relational engine (e.g. SQLite → SQL Server). DataPortStudio maps each column to a compatible type, creates the table, and copies the rows. Large SQL Server copies use high-speed bulk streaming.

Clarion → SQL: review & tweak the column types

When you copy a Clarion TPS or DAT file into a SQL database, DataPortStudio shows a column mapping dialog before creating the table:

  • Each source column is listed with its Clarion type and a proposed SQL type.
  • Pick the SQL type from a dropdown (specific to the target engine) and set a size / precision where it applies (length for text, precision,scale for decimals).
  • Columns that look like Clarion dates/times are pre-mapped to SQL date / time, and their values are converted on copy (not dumped as raw numbers).
  • Use Reset to suggested to restore the automatic mapping, then Apply & copy.
Don't want a date/time converted? Just set that column back to int in the dialog and the raw Clarion number is copied instead.

Generate INSERT, Import & Export

  • Generate INSERT (right-click a SQL Server table) produces an INSERT script for its rows, which you can view, copy or save as .sql.
  • Import data loads rows into a table from a file.
  • Export the current table or query results to CSV, TXT, JSON, XML, HTML, Excel (XLSX / legacy XLS), DBF or a SQL script. Choose the filter scope and formatting in the export dialog, and open the file or its folder when done.

Users & roles

The Users button on the command bar opens the User Manager for the selected connection (available on SQL Server, MySQL/MariaDB). You can list users and roles, create/drop them, set or change passwords, and manage privileges — server logins/roles and permissions on SQL Server; user@host accounts, global and per-database privileges, and roles on MySQL 8 / MariaDB.

Settings

File ▸ Settings lets you choose the language (English / Spanish), the default row limit for opening tables, and appearance options. Changes apply immediately.

Frequently asked questions

1. Do I need to install DataPortStudio or .NET?

No. DataPortStudio is a single portable DataPortStudio.exe — just download and run it. It's self-contained, so no .NET installation is required. The first launch is a couple of seconds slower while it self-extracts.

2. Where are my connections and passwords stored?

Connections are saved on your PC and restored each time you start DataPortStudio. Passwords are encrypted with Windows data protection (tied to your Windows user account), not stored in plain text.

3. Why can't I add or delete rows in a TPS or DAT table?

Clarion TPS supports cell edits (UPDATE) — click a cell, change the value, and save. Adding and deleting rows is not supported because those operations require rebuilding the TPS index files (.K??/.I??), which DataPortStudio does not manage. Clarion DAT and MongoDB remain fully read-only.

4. How do I switch between English and Spanish?

Open File ▸ Settings and choose the language. The interface updates live (no restart), and this User Guide opens in whichever language you've selected.

5. My table only shows part of the rows — why?

Tables open with a row limit so large tables load quickly. Raise the default limit in File ▸ Settings, or apply a filter to narrow down to the rows you need.

6. How do I move a Clarion TPS/DAT file into SQL Server?

Create a Clarion connection pointing at the folder, right-click the file ▸ Copy, then Paste onto the target database's Tables folder. Review the proposed SQL types in the mapping dialog, adjust if needed, and click Apply & copy.

7. My Clarion dates/times show as numbers — how do I fix that?

Clarion stores dates and times as integers. DataPortStudio detects those columns and can display them as readable dates/times — use the toggle on the table toolbar. When copying to SQL, those columns are pre-mapped to date/time and the values are converted automatically.

8. Can I copy a table between two different database types?

Yes. You can copy between relational engines (for example SQLite → SQL Server, Firebird → MySQL, or PostgreSQL → SQLite) and from Clarion files into any SQL database. DataPortStudio maps each column to a compatible type and copies the rows.

9. When I edit a table it warns there's "no primary key" — what do I choose?

Without a primary key, DataPortStudio needs to know which column(s) uniquely identify a row so edits and deletes affect the right one. Pick a column (or set) that is unique, or keep the default "all columns", which is the safest option.

10. How do I update DataPortStudio?

Download the newest DataPortStudio.exe from the Help ▸ Latest Release page and replace your copy. Your saved connections and settings are kept. You can check your current version in Help ▸ About.
